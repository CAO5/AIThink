# AIThink 智谱清言自动登录操作手册

## 1. 项目简介

AIThink 是一个高性能的 AI 工具 API 服务，支持通过浏览器自动化模拟登录智谱清言等 AI 平台。

### 核心特性

- ✅ **自动登录**：首次手动登录后，后续自动登录
- ✅ **高性能**：基于 Go + Gin 框架，支持高并发
- ✅ **会话管理**：支持多账号管理
- ✅ **RESTful API**：标准的 HTTP 接口
- ✅ **AI视觉识别**：智能分析页面，解决网页动态变更问题（New！）
- ✅ **配置管理**：提供API接口管理AI视觉服务配置

## 2. 环境要求

### 必需环境

1. **Go 1.21+**
   - 下载：https://go.dev/dl/
   - 验证：`go version`

2. **Chrome/Chromium 浏览器**
   - Windows：安装 Google Chrome
   - 路径：`F:\Program Files (x86)\Go\bin\go.exe`（已安装）

3. **系统依赖**
   - Windows 10+
   - 内存：建议 4GB 以上

## 3. 安装步骤

### 3.1 下载依赖




## 4. 使用指南

### 4.1 首次登录（需要手动输入验证码）

#### 4.1.1 调用登录接口

```powershell
$loginBody = @{
    platform = "zhipu"
    username = "你的手机号"
    password = "123456"  # 占位符，实际不用
    user_data_dir = "sessions/zhipu_main"  # 指定用户数据目录
} | ConvertTo-Json

Invoke-RestMethod -Uri http://localhost:8081/api/v1/login `
    -Method Post `
    -Body $loginBody `
    -ContentType "application/json"
```

#### 4.1.2 手动完成登录

1. 程序会打开 Chrome 浏览器窗口
2. 在浏览器中输入手机号和验证码
3. 完成登录后，程序会自动检测登录状态
4. 登录成功后，Cookie 会自动保存到 `sessions/zhipu_main` 目录

**预期返回：**

```json
{
  "code": 0,
  "message": "登录成功",
  "data": {
    "session_id": "zhipu_1234567890",
    "message": "登录成功"
  }
}
```

### 4.2 后续自动登录

使用相同的 `user_data_dir` 再次调用登录接口：

```powershell
$loginBody = @{
    platform = "zhipu"
    username = "你的手机号"
    password = "123456"
    user_data_dir = "sessions/zhipu_main"  # 与首次登录相同
} | ConvertTo-Json

Invoke-RestMethod -Uri http://localhost:8081/api/v1/login `
    -Method Post `
    -Body $loginBody `
    -ContentType "application/json"
```

**效果：**
- Chrome 会自动加载保存的登录状态
- 无需重新输入验证码
- 实现真正的自动登录

### 4.3 向智谱清言提问

```powershell
$askBody = @{
    platform = "zhipu"
    session_id = "上一步返回的session_id"
    question = "你好，请介绍一下你自己"
} | ConvertTo-Json

Invoke-RestMethod -Uri http://localhost:8081/api/v1/ask `
    -Method Post `
    -Body $askBody `
    -ContentType "application/json"
```

**预期返回：**

```json
{
  "code": 0,
  "message": "提问成功",
  "data": {
    "answer": "我是智谱清言，是由智谱AI公司开发的...",
    "session_id": "zhipu_1234567890"
  }
}
```

### 4.4 登出

```powershell
Invoke-RestMethod -Uri "http://localhost:8081/api/v1/logout?session_id=你的session_id" `
    -Method Post
```

## 5. 工作原理

### 5.1 自动登录机制

```
┌─────────────────────────────────────────────────────┐
│                  自动登录流程                        │
├─────────────────────────────────────────────────────┤
│  1. 首次登录                                       │
│     ├─ 创建浏览器会话（指定user_data_dir）          │
│     ├─ 打开浏览器，手动输入验证码                   │
│     ├─ 完成登录                                   │
│     └─ Chrome自动保存Cookie到user_data_dir         │
│                                                     │
│  2. 后续登录                                       │
│     ├─ 使用相同的user_data_dir创建会话             │
│     ├─ Chrome自动加载保存的Cookie                  │
│     ├─ 访问聊天页面验证登录状态                    │
│     └─ 自动登录成功，无需验证码                    │
└─────────────────────────────────────────────────────┘
```

### 5.2 技术实现

- **Chrome 用户数据目录**：`--user-data-dir` 参数
- **Cookie 持久化**：Chrome 原生支持
- **登录状态检测**：访问页面并检查元素

## 8. 常见问题

### Q1: 自动登录失败怎么办？

**原因：** Cookie 过期或被清除

**解决：**
```powershell
# 删除旧的用户数据目录
Remove-Item -Recurse -Force "sessions/zhipu_main"

# 重新手动登录一次
# 然后后续就可以自动登录了
```

### Q2: 提示"端口已被占用"

**解决：**
```powershell
# 查看占用端口的进程
netstat -ano | findstr :8081

# 结束进程（假设PID是12345）
taskkill /PID 12345 /F
```

或修改端口：
```powershell
$env:PORT="8082"
./bin/aithink.exe
```

### Q3: 浏览器没有自动打开

**检查：**
1. Chrome 是否已安装
2. 程序是否有足够权限
3. 查看日志输出

### Q4: 验证码收不到

**可能原因：**
1. 手机号输入错误
2. 网络问题
3. 平台限制（短时间内请求过多）

**建议：** 等待一段时间后重试

### Q5: 如何查看详细日志

程序运行时会输出详细日志：
- `创建浏览器会话: xxx`
- `尝试自动登录...`
- `自动登录成功！`
- `开始手动登录流程...`
- `点击获取验证码按钮`
- `请在浏览器中输入验证码完成登录...`

## 9. AI视觉识别功能（New！）

AIThink现在支持使用AI视觉识别技术来智能分析网页，解决网页动态变更导致选择器失效的问题。

### 9.1 工作原理

当系统执行自动化操作时：

1. **常规方法优先**：首先尝试使用JavaScript和DOM查询查找元素
2. **AI后备方案**：如果常规方法失败，自动使用AI视觉识别
3. **截图分析**：对当前页面截图并发送给配置的AI服务
4. **智能操作**：使用AI返回的选择器进行操作

### 9.2 支持的AI服务

- **OpenAI GPT-4V**（推荐）：识别准确度高
- **百度OCR**：国内访问快
- **腾讯云OCR**：企业级服务
- **自定义API**：接入任何支持图片分析的API

### 9.3 配置AI视觉服务

#### 9.3.1 获取当前配置

```powershell
Invoke-RestMethod -Uri http://localhost:8081/api/v1/config -Method GET
```

#### 9.3.2 配置OpenAI GPT-4V（推荐）

```powershell
$body = @{
    provider = "openai"
    openai = @{
        api_key = "sk-your-openai-key"
        base_url = "https://api.openai.com/v1"
        model = "gpt-4-vision-preview"
    }
} | ConvertTo-Json

Invoke-RestMethod -Uri http://localhost:8081/api/v1/config -Method POST -ContentType "application/json" -Body $body
```

**注意**：
- 需要有效的OpenAI API Key
- GPT-4V模型需要申请访问权限
- 按token计费，建议合理使用

#### 9.3.3 配置百度OCR

```powershell
$body = @{
    provider = "baidu"
    baidu = @{
        api_key = "your-baidu-api-key"
        secret_key = "your-baidu-secret-key"
    }
} | ConvertTo-Json

Invoke-RestMethod -Uri http://localhost:8081/api/v1/config -Method POST -ContentType "application/json" -Body $body
```

#### 9.3.4 配置腾讯云OCR

```powershell
$body = @{
    provider = "tencent"
    tencent = @{
        secret_id = "your-secret-id"
        secret_key = "your-secret-key"
    }
} | ConvertTo-Json

Invoke-RestMethod -Uri http://localhost:8081/api/v1/config -Method POST -ContentType "application/json" -Body $body
```

### 9.4 在登录流程中的应用

配置完成后，登录流程会自动使用AI视觉识别：

```powershell
$loginBody = @{
    platform = "zhipu"
    username = "13800138000"
    password = ""
} | ConvertTo-Json

Invoke-RestMethod -Uri http://localhost:8081/api/v1/login `
    -Method Post `
    -Body $loginBody `
    -ContentType "application/json"
```

**工作流程：**

1. 系统打开智谱清言网站
2. 尝试常规方法查找手机号输入框
3. 如果失败，自动截图并使用AI视觉识别
4. AI返回建议的选择器
5. 使用AI提供的选择器进行输入和点击操作

### 9.6 优势

1. **适应网页变化**：不依赖固定的选择器，AI可以识别页面元素的语义
2. **提高健壮性**：多重保障（常规方法 + AI后备）
3. **易于扩展**：支持多种AI服务，可以轻松添加新的AI服务商

### 9.7 注意事项

1. **API Key安全**：配置文件中不返回完整的API Key（只告知是否已配置）
2. **性能考虑**：AI视觉识别比传统方法慢（需要网络请求和AI处理）
3. **成本考虑**：OpenAI GPT-4V按token计费，建议只对关键操作使用AI识别
4. **准确性**：AI识别不是100%准确，建议保存截图和AI响应便于调试

## 11. 注意事项

1. **验证码处理**：首次登录需要手动输入验证码，这是平台的安全机制
2. **登录状态有效期**：Cookie 会过期，过期后需要重新登录
3. **并发限制**：虽然支持高并发，但请遵守智谱清言的使用限制
4. **浏览器资源**：每个会话一个浏览器实例，注意服务器资源
5. **数据隐私**：用户数据目录包含登录信息，请妥善保管
